# NetVis v0.6.5 — weight-blob format fixes (issue #85, frozen plan)

> **Status: contracts frozen by the lead.** This finishes the two weight-blob
> parsers from v0.5.0 for the on-disk layouts real models actually ship in:
> OpenVINO IR whose sibling `.bin` is missing/quantized, and modern Apple
> `.mlpackage` (mlProgram / MIL) bundles whose weights live in an external
> `weights/weight.bin`. It **extends** the existing parsers — it does not
> reinvent them. Every research number below is verified against Apple
> `coremltools` `main` source (see §Research provenance).

## Non-negotiable invariants (unchanged from the product thesis)

1. **Zero tensor-payload reads during structural parse** — `parse()` records
   weights as `TensorRef{file_offset, byte_len}` only; `ByteReader::payload_read_counter()`
   stays `0`. The single accounted payload read is in `TensorStats.cpp`.
2. **Hostile-input-safe** — every offset/len read through the bounds-checked
   `WireReader`/`XmlDocument`/`ByteReader`; malformed input → `Result` error with
   a byte offset, never a crash. Directory walking is path-traversal-guarded.
3. **`ir::DType` is FROZEN at 16 enumerators, `Unknown=15`.** Phase B's plugin SDK
   `static_assert`s this exact layout (`docs/v0.6.x-followups-plan.md` §D.4). **No
   new DType may be added here.** Exotic quant types (i4/u4/nf4/u1, fp8, sub-byte)
   ride as a *label*, not a DType (see §Contract C1).
4. **No `LayoutCache::kVersion` bump** — `TensorRef` is not in `structure_hash`
   (node fingerprint hashes op_type/arity/attributes only, `CollapseTree.cpp:190`)
   and is never serialized to the layout cache. The two new `TensorRef` fields are
   layout-inert by construction.
5. **Parsers stay pure `Result<ir::Model> parse(const MappedFile&, ProgressSink&)`.**
   The `.mlpackage`-is-a-directory problem is solved at the OPEN layer, above the
   parser, so `parse()` always receives one mmap of one file (§Contract C2).

## The contract (LEAD writes all of this before any agent runs)

### C1 — `ir::TensorRef` gains two trailing, additive fields (`ir/IR.h`)

```cpp
struct TensorRef {
  StringId name;
  DType dtype = DType::Unknown;
  SmallVec<int64_t, 6> shape;
  uint64_t file_offset = UINT64_MAX;
  uint64_t byte_len = 0;
  StringId external_path;
  // v0.6.5 (#85), APPEND-ONLY:
  StringId dtype_label;      // human dtype when the exact type has no ir::DType
                             // (OpenVINO i4/u4/nf4/u1/f8*, MIL sub-byte). Empty
                             // otherwise. Display falls back to dtype_name(dtype).
  bool blob_indirect = false;// true => file_offset points at a CoreML MIL
                             // blob_metadata header (64B, sentinel 0xDEADBEEF),
                             // NOT raw data. TensorStats follows the header.
  // ... elem_count() unchanged ...
};
```

Rationale: keeps `DType` frozen (inv 3) while letting the inspector *label* an
exotic tensor honestly. `blob_indirect` moves the MIL header indirection to the
one sanctioned payload reader instead of the parser.

### C2 — bundle resolution at the OPEN layer (`src/engine/ModelPath.{h,cpp}`, NEW)

```cpp
// engine/ModelPath.h
namespace netvis {
struct ResolvedModelPath {
  std::string display_path;  // what the user opened (the .mlpackage dir, or the file)
  std::string map_path;      // the single file to mmap (inner model.mlmodel for a bundle)
};
// If `path` is a .mlpackage directory: read Manifest.json, resolve
// rootModelIdentifier -> itemInfoEntries[uuid].path -> Data/<path>; fall back to
// the conventional Data/com.apple.CoreML/model.mlmodel if the manifest is
// unreadable. Path-traversal guarded (reject any resolved path that escapes the
// bundle root). Otherwise map_path == display_path == path.
ResolvedModelPath resolve_model_path(const std::string& path);
}
```

Wiring (LEAD):
- `ModelSession` gains `std::string map_path_;`. `open_async` sets
  `path_ = r.display_path; map_path_ = r.map_path;` and maps `map_path_`.
  `model_dir()` returns `parent_path(map_path_)` — so for a bundle it points at
  `Data/com.apple.CoreML/`, and the existing `resolve_payload` external-path
  plumbing (`TensorStats.cpp:47`) resolves `weights/weight.bin` with **zero new
  code**.
- `DiffLoader::load_comparison` calls `resolve_model_path(path)` and maps
  `r.map_path` (diff reads topology only; model-B external weights are a non-goal).
- `Detect.cpp`: no change required — the inner file is `model.mlmodel` (ext
  `mlmodel`) and already routes to CoreML.

### C3 — `TensorStats` follows the MIL blob header when `blob_indirect` (`TensorStats.cpp`)

`resolve_payload` already opens the external file + bounds-checks `[off, off+len)`.
When `t.blob_indirect`, after resolving to the header offset it must (all through
the bounds-checked path, still exactly one `mark_payload_read()`):
1. read the 64-byte `blob_metadata` at `file_offset`; validate `sentinel == 0xDEADBEEF`
   (else `err`); read `mil_dtype (u32)`, `sizeInBytes (u64)`, `data_offset (u64)`.
2. re-point the payload view to `[data_offset, data_offset + sizeInBytes)`,
   bounds-checked against the external file size.
This is the security-sensitive surface (hostile blob) → LEAD writes it.

### C4 — fixtures are part of the frozen contract (LEAD writes ALL of `tools/gen_fixtures.py` additions)

The `.mlpackage` fixture (with a real 64-byte `blob_metadata` header) IS the spec
the CoreML agent implements against, so the lead authors it to guarantee both the
parser agent and the test agent build against identical bytes. LEAD adds:
- `build_coreml_mlprogram()` + a `.mlpackage` **directory** fixture:
  `tests/fixtures/model.mlpackage/{Manifest.json, Data/com.apple.CoreML/model.mlmodel, Data/com.apple.CoreML/weights/weight.bin}`.
- `build_openvino_quant_xml()` / a missing-`.bin` variant fixture for OpenVINO.

## Research provenance (verified against coremltools `main`)

- Bundle: `Data/com.apple.CoreML/model.mlmodel` + `weights/weight.bin`;
  `Manifest.json` → `rootModelIdentifier` → `itemInfoEntries[uuid].path`
  (= `<author>/<name>`, prepend `Data/`). (`modelpackage/src/ModelPackage.cpp`,
  `coremltools/models/utils.py`.)
- `Model.mlProgram` = field **502**, holds `MILSpec.Program`. (`Model.proto`.)
- MIL field numbers (`MIL.proto`): `Program{version=1, functions=2(map), attributes=4}`;
  `Function{inputs=1, opset=2, block_specializations=3(map)}`;
  `Block{inputs=1, outputs=2, operations=3}`;
  `Operation{type=1, inputs=2(map), outputs=3, blocks=4, attributes=5}`;
  `NamedValueType{name=1, type=2}`; `Argument{arguments=1}`, `Binding{name=1, value=2}` (oneof);
  `Value{type=2, immediateValue=3, blobFileValue=5}` (**field 4 skipped**);
  `BlobFileValue{fileName=1, offset=2(uint64)}`;
  `ValueType{tensorType=1,...}`; `TensorType{dataType=1, rank=2, dimensions=3(repeated)}`;
  `Dimension{constant=1->size(u64)=1, unknown=2}`.
- MIL proto `DataType` enum (distinct from blob enum!): FLOAT16=10, FLOAT32=11,
  FLOAT64=12, BFLOAT16=13, INT8=21, INT16=22, INT32=23, INT64=24, INT4=25,
  UINT8=31..UINT64=34, UINT4=35, sub-byte 36-39, FLOAT8E4M3FN=40, FLOAT8E5M2=41,
  BOOL=1, STRING=2.
- `weight.bin` **blob v2**: `storage_header` 64B `{count u32@0, version u32@4(=2), 7×reserved u64}`;
  `blob_metadata` 64B `{sentinel u32@0 =0xDEADBEEF, mil_dtype u32@4, sizeInBytes u64@8, data_offset u64@16, padding_bits u64@24, 4×reserved}`;
  `DefaultStorageAlignment=64`. `BlobFileValue.offset` → metadata header;
  raw data at `metadata.data_offset`. Do NOT validate reserved fields (garbage pre-iOS18).
  Blob `BlobDataType` enum (distinct): Float16=1, Float32=2, UInt8=3, Int8=4, BFloat16=5, ...
- proto3 `map<K,V>` on the wire = repeated 2-field entry messages `{key=1, value=2}`.
