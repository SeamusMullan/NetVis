# NetVis v0.6.x — roadmap: follow-ups, out-of-scope, and backlog

> Status: **planning**. v0.6.0 (plugins) is **released** (registry spine +
> declarative DSL + WASM sandbox; #7/#8/#9/#12 closed, #10/#11 partial). This
> document is the authoritative map of what comes next: the plugin follow-ups that
> finish #10/#11, the items deliberately **out of scope** (with the reason and what
> it would take), and the broader feature backlog grouped by theme.

---

## Part A — Plugin follow-ups (finish #10 / #11)

Detailed implementable designs live in `docs/v0.6.x-followups-plan.md`. Summary:

| Increment | Closes | Scope | Risk |
|---|---|---|---|
| WASM OpHandler adapter | #10 | WASM-backed category/color/flops/infer_shape (declarative-escalation slow path); host imports to read an OpContext without byte access; category clamp | medium |
| WASM ParserPlugin + host parser API | #10 | `host_read_range` (structure bytes only, rejects tensor-range overlap), `host_record_tensor` (offset+len), `host_add_node/value/attr`; hooks Detect's `Format::Unknown` branch | **high** (thesis surface) |
| Plugin enable/disable persistence + WASM trust gate | #11 | `"plugins"` object in `view_prefs.json` (declarative on, WASM off by default); loader consults the enable-set before registering; per-plugin confirm dialog | low |
| SDK header + `plugins/examples/` + `docs/plugin-abi.md` | #10/#12 | `plugins/sdk/netvis_plugin.h` (C freestanding), example toy_parser/attn_pass sources, ABI doc, a static_assert bridge keeping the SDK DType enum in sync with `ir::DType` | low |

**Sequencing:** freeze any new host imports first (`g++ -fsyntax-only`), then the
WASM OpHandler + WASM parser are independent, then persistence/trust (UI), then the
SDK (documents the frozen ABI). Verification bar unchanged: 185-test baseline +
ASan/UBSan/TSan, and — the hard lesson from the v0.6.0 tag — **validate the
macOS+Windows package build via a `release-override.yml` draft dispatch BEFORE
tagging** (the tag-only package matrix is the only place cross-OS compile is
exercised; a numeric `X.Y.Z` version is required or CMake `project(VERSION)` fails
configure).

## Part B — Out of scope (design non-goals) — reason + what it would take

These are **deliberately excluded**, carried from `docs/v0.6.0-plan.md` and refined.
Each is a conscious invariant-driven decision, not an oversight.

| Item | Why out of scope | What it would take to reconsider |
|---|---|---|
| **View-panel / custom ImGui plugins** | A panel needs either native code (breaks the sandbox + single-binary + zero-payload invariants) or a full retained-mode UI-description protocol marshalled over WASM (a large separate design). | A declarative UI-description protocol (widget tree → host renders) over the existing WASM boundary, OR a native-behind-a-C-ABI-shim path for explicitly-trusted plugins only. Large; revisit post-1.0. |
| **Native `.so` / `.dll` plugins** | No sandbox (arbitrary in-process code); unenforceable zero-payload (raw mmap access); C++ ABI fragility across compiler/stdlib; per-arch/per-compiler artifacts break the single-`.exe` distribution. | Rejected as a primary mechanism. Everything reachable via declarative + WASM. Only a panel-plugin need could ever justify a narrow trusted-only C-ABI shim. |
| **Dequantization / payload-transforming plugins** | Directly breaks the zero-payload thesis — a plugin that decodes weights defeats the invariant that makes NetVis open multi-GB models instantly. | Never, by thesis. A *view-only* dequant PREVIEW (issue #49) is a host feature gated behind the existing single decode job, not a plugin capability. |
| **Per-node WASM as a routine pattern** | Interpreter + marshalling cost per node is a footgun on a 100k-node graph. | Allowed as a documented declarative-escalation slow path with a perf-guard warning; the fast per-node path stays declarative closures. |
| **wasmtime backend** | wasm3 keeps the vendored single-static-binary story with no LLVM/codegen dep. | The host-API boundary is already runtime-agnostic (`WasmRuntime` wrapper), so wasmtime is a compile-time drop-in if a speed need arises. Not now. |
| **WASI / syscalls in the sandbox** | Zero-syscall is the sandbox guarantee; the WASI translation units are excluded from the wasm3 build. | Never — the capability host API is the only surface; a new capability is a deliberate, reviewed host import, not WASI. |
| **`.mlpackage` directory bundles** (CoreML) | Directory-open is a `ModelSession`/`File→Open` change, not a parser change. | A session/UI change to accept a directory + resolve the bundle manifest → route the inner `.mlmodel` + `weights/weight.bin`. Modest; a v0.7 candidate. |
| **HDF5 chunked-dataset payload stitching** (Keras) | Chunked layout is a B-tree of chunks, not a single linear range; stitching contradicts offset+len. | Record shape/dtype with `file_offset=UINT64_MAX` + a note (current honest fallback). A read-time chunk iterator behind the single decode job could preview one chunk. |
| **`np.savez_compressed` payload addressing** | Deflated entries aren't linearly mmap-addressable. | Same honest fallback (header known, offset `UINT64_MAX`). |
| **Full TorchScript graph reconstruction** | Remains the v1 non-goal; #40 shipped an op *inventory*, not a compute graph. | Would need a TorchScript IR reader (bytecode + type tables) — a large parser, not a v0.6.x item. |
| **General XML / HDF5 parsers** | General parsers are a dependency + a hostile-input minefield; NetVis ships only the bounded structural subset each format needs. | Never as a general dep. Each format's bounded reader is extended in place if a real file needs more. |

## Part C — Feature backlog (open issues, grouped) — NOT plugins

Themed backlog for v0.7+. Each is an independent feature; none depends on the
plugin milestone. Priority is indicative.

### Graph UX (#13–#22)
Clickable op-type legend (#13), critical-path highlight (#14, ties to analyzer
FLOP/latency), path-between-two-nodes (#15), named subgraph bookmarks (#16),
breadcrumb trail (#17), edge shape/dtype hover tooltip (#18), sticky pinned nodes
(#19), depth ruler / layer bands (#20), collapse-all/expand-to-depth-N (#21),
orthogonal edge routing toggle (#22). Mostly view-layer; #14 reads the CostReport.

### Analyzer coverage + tooling (#1–#6, #25–#32)
Op coverage: MatMulNBits (#1), contrib ops QAttention/GroupQueryAttention/
Skip+EmbedLayerNorm/DynamicQuantizeMatMul/MatMulIntegerToFloat (#2), variadic
Sum/Mean fan-in factor (#3), MHA packed-QKV rank-5 + cross-attn dynamic-key (#6) —
**these are now prime declarative/WASM-plugin candidates** (the plugin system exists
to add op coverage without a release). Roofline ridge presets cite sources +
user-tunable ridge (#4). Tooling: top-N cost hotspots (#25), cumulative-cost
sparkline (#27), what-if batch-size slider (#29), custom machine-balance presets
editor (#30), two-model cost diff (#31, ties to diff), receptive-field per node
(#32). View-layer cost cache-keying headless harness (#5 — the documented test gap).

### Diff (#33–#37)
Summary panel with added/removed/changed + clickable list (#33), per-tensor
weight-stat diff (#34), match-by-name vs match-by-topology toggle (#35), 3+ model
comparison (#36), export change report md/TSV (#37).

### Weight inspector (#44, #46–#51)
GGUF metadata (KV) panel (#44), per-channel stat breakdown (#46), 2D weight heatmap
thumbnail (#47), outlier/dead-channel flags (#48), **GGUF quant-block dequant
preview (#49 — view-only, behind the single decode job, NOT a plugin**), compare
same-named tensor across two models (#50), value search within a tensor (#51). All
route through the existing single payload-decode job — the ONE place bytes are read.

### Search (#52–#54)
Regex + field-scoped queries `op:Conv name:layer.*` (#52), results-list panel (#53),
query by shape/dtype/param-count (#54).

### Export (#55–#58)
SVG/PDF vector export of the current view (#55), shareable view-state file (#56),
copy node as JSON (#57), **headless report CLI `netvis --report model.onnx → JSON`
(#58 — a natural pairing with the PassPlugin metrics + CostReport; a strong v0.7
headline**).

### UX (#59–#62)
Command palette Ctrl+P (#59), attribute-inspector filter/search (#60), async
first-layout progress + cancel for 100k-node graphs (#61), multi-model tabs (#62).

### Format (#67)
Nexus model manifest (DirektDSP) — a workspace-index view, not a format parser;
deferred until after the plugin follow-ups (a Nexus reader could ship as a WASM
ParserPlugin once the parser host API lands).

## Suggested v0.7 theme

Two coherent candidates, both leveraging what v0.6.0 built:
1. **"Report & share"** — headless report CLI (#58) + view-state file (#56) + copy
   node JSON (#57) + diff export (#37), all reading the CostReport/PassPlugin
   surface the plugin milestone finalized.
2. **"Op coverage via plugins"** — ship the analyzer op-coverage gaps (#1/#2/#3/#6)
   AS declarative/WASM plugins in `plugins/examples/`, proving the plugin system
   pays for itself and closing coverage without bloating the core.
