# NetVis v0.6.0 — PLUGIN milestone (frozen design)

> Status: **frozen / contracts freezing**. This document is the authoritative spec the implementation phase builds against. It merges the 5-facet design workflow output with the adjudicated fixes from the adversarial critique pass (every facet came back `needs-revision`; the real defects the critics found are folded in below and marked **[critique-fix]**). It corrects `docs/v0.6.0-plan.md` wherever the sketch disagrees with the real `src/` code; those corrections are marked **[plan-correction]**.

## Goal

Turn NetVis's fixed op/parser/analysis surface into a **plugin surface** without breaking any of the six invariants. Three plugin backends land across three increments, all behind one frozen registry spine:

1. **Op handlers** — replace/extend per-op category, color, FLOPs, and output-shape inference (the `compute_flops` / `categorize_op` / `infer_shapes_ext` work).
2. **Parser plugins** — add a new file format (the `parse(const MappedFile&, ProgressSink&)` + detect surface).
3. **Analysis passes** — compute named metrics over a parsed `Model` + its `CostReport`.

Two implementations of each: a safe-by-construction **declarative** backend (JSON manifest + integer expression DSL) and a sandboxed **WASM** backend (wasm3), both dogfooding the same registry the built-ins register into.

## Non-negotiable invariants (every part upholds all six)

1. Single self-contained binary — vendored-source-only, static single-exe.
2. Cross-platform (Win/Mac/Linux).
3. **Zero tensor-payload reads** — parsers/handlers record offset+len only; enforced by `ByteReader::payload_read_counter()==0` in every parser test.
4. Hostile-input-safe — bounds-checked, bounded depth/alloc, no unbounded recursion.
5. Never throw / never crash — return `core::Result<T>` with a byte offset, or a partial/labelled model.
6. Honest-unknown — unsupported op / unresolved shape ⇒ `flops_known=false`, excluded from totals, never fabricated.

## Part 0 — Cross-facet contradictions, resolved once

These are the load-bearing decisions where facets disagreed. Each is settled here and referenced everywhere else.

| # | Contradiction | **Frozen answer** |
|---|---|---|
| 0.1 | Manifest format: TOML (plan lean) vs JSON | **JSONC via the already-linked `nlohmann/json`** (§Q1). No new dependency, no new hostile-input surface. |
| 0.2 | DSL input addressing: TOML aliases (`key="in1"`) vs slot indices | **Slot indices only** (`in0..inN`, `out0..outN`, `O`). `compute_flops` addresses inputs by slot (Conv weight=slot 1, QLinearConv=slot 3), so the DSL speaks slots natively. |
| 0.3 | Dynamic/`-1` dim propagation | **FLOPs**: any dim `<1` read ⇒ `flops_known=false`. **Shape**: unknown dim ⇒ `-1`; write the shape if ≥1 dim resolves; skip only if *all* unknown. Mirrors `elem_count_from_shape` (`d<1 ⇒ 0`) and `infer_shapes_ext` honest-unknown. |
| 0.4 | Attr semantics: `std::optional` (unknown-on-absent) vs defaulted getter | **OpContext exposes BOTH**: `attr_int(name, default)` (mirrors `get_int_attr` exactly, returns default on absent-or-wrong-kind) AND `has_attr(name)`. | 0.5 | Override precedence: backend ladder vs enumeration order | **Total order = backend tier PRIMARY, enumeration order TIEBREAKER within a tier.** WASM > declarative > built-in; within a tier, first in deterministic path-sorted enumeration wins. **[critique-fix]** |

| 0.7 | WASM zero-payload "enforced by counter==0" | **Counter is honor-system; the HOST owns marking.** `host_read_range` overlapping any recorded tensor range is rejected (or increments the counter). Enforcement moves into the host read boundary. **[critique-fix]** |

### 2.0 Prerequisite refactor: promote the saturating math out of `CostModel.cpp` **[critique-fix]**

`safe_add`/`safe_mul`/`elem_count_from_shape`/`partial_shape_product` are TU-local `static` in `CostModel.cpp`. The plan requires plugin arithmetic to route through *the same* saturating primitives; that is impossible while they are TU-local. Extract them into pure leaf headers, then have `CostModel.cpp` `#include` and use them (behavior-preserving — same definitions), so built-in and plugin FLOPs are bit-identical. `partial_shape_product`'s bound is **hardened** against uint32 wrap. **[critique-fix]**

```cpp
// core/SafeMath.h — pure, <cstdint> only. Extracted verbatim from CostModel.cpp,
// plus SIGNED variants the DSL needs (it has subtraction; the built-ins never do).
#pragma once
#include <cstdint>
namespace netvis {

// --- unsigned saturating (the built-in FLOP path; clamp at UINT64_MAX) --------
inline uint64_t safe_add(uint64_t a, uint64_t b) {
  if (UINT64_MAX - a < b) return UINT64_MAX;
  return a + b;
}
inline uint64_t safe_mul(uint64_t a, uint64_t b) {
  if (a == 0 || b == 0) return 0;
  if (UINT64_MAX / a < b) return UINT64_MAX;
  return a * b;
}

// --- signed checked int64 (the DSL domain) ------------------------------------
// Return false on overflow/UB; the DSL treats any false here as honest-unknown.
// Guards INT64_MIN/-1 (div/rem overflow UB) and add/sub/mul overflow.
bool checked_add_i64(int64_t a, int64_t b, int64_t* out);
bool checked_sub_i64(int64_t a, int64_t b, int64_t* out);
bool checked_mul_i64(int64_t a, int64_t b, int64_t* out);
bool checked_div_i64(int64_t a, int64_t b, int64_t* out);   // false if b==0 || (a==INT64_MIN && b==-1)
bool checked_mod_i64(int64_t a, int64_t b, int64_t* out);   // same guard as div

}  // namespace netvis
```

```cpp
// engine/plugin/ShapeMath.h — pure; includes core/SmallVec.h only.
#pragma once
#include "core/SmallVec.h"
#include "core/SafeMath.h"
namespace netvis::plugin {
using Shape = SmallVec<int64_t, 6>;

// Product of dims; 0 if any dim < 1 (== CostModel.cpp elem_count_from_shape).
inline uint64_t elem_count(const Shape& s) {
  if (s.empty()) return 0;
  uint64_t n = 1;
  for (int64_t d : s) { if (d < 1) return 0; n = safe_mul(n, (uint64_t)d); }
  return n;
}

// Product of dims [off, off+count); 0 if any dim < 1. Bound is wrap-proof:
// NO addition that can overflow uint32 (the CostModel.cpp form `off+count>size`
// wraps when count is computed as size-2 on a <2-length shape). [critique-fix]
inline uint64_t partial_product(const Shape& s, uint32_t off, uint32_t count) {
  const uint32_t n = (uint32_t)s.size();
  if (count > n || off > n - count) return 0;   // no wrapping add
  uint64_t p = 1;
  for (uint32_t i = 0; i < count; ++i) {
    int64_t d = s[off + i];
    if (d < 1) return 0;
    p = safe_mul(p, (uint64_t)d);
  }
  return p;
}
}  // namespace netvis::plugin
```

### 2.1 `engine/plugin/OpHandler.h` — the pure op-extension ABI (crux)

```cpp
// engine/plugin/OpHandler.h — FROZEN op-level plugin ABI (v0.6.0).
//
// An OpHandler answers "what is this op": color category, optional explicit color,
// FLOP estimate, and output-shape propagation — from STRUCTURE ONLY. OpContext has
// NO byte-access term, so the zero-payload thesis (invariant 3) is a property of
// the TYPE, not a convention: a handler literally cannot read a weight. The one
// bytes-adjacent accessor (input_const_ints) is HOST-mediated, bounded to <=64
// I32/I64 metadata elems, and never bumps the payload counter — see below.
//
// This mirrors compute_flops() in CostModel.cpp, the reference implementation
// every built-in handler ports from. PURITY: includes ir/IR.h + engine leaf
// headers only — identical include set to CostModel.cpp. No view/ImGui/parser dep.
#pragma once

#include <cstdint>
#include <optional>
#include <string_view>
#include <vector>

#include "core/SmallVec.h"
#include "engine/HeatmapGradient.h"   // Rgba8 (leaf: <cstdint> only)
#include "engine/OpCategory.h"        // OpCategory, categorize_op
#include "engine/plugin/ShapeMath.h"  // Shape, elem_count, partial_product
#include "ir/IR.h"                    // ir::DType, ir::Model, StringId

namespace netvis::plugin {

// Payload-format ABI version for DSL manifests + WASM modules. NOTE: this is NOT
// a C++ binary-vtable version — invariant 1 means every handler (built-in, DSL
// closure, WASM adapter) is compiled INTO the single binary as one unit, so there
// is no dlopen of an external C++ .so and vtable layout is never crossed. This
// constant gates (a) declarative manifest `api_version` and (b) the WASM module's
// exported netvis_abi_version(); a mismatch => that plugin is refused, never
// half-registered. [invariant-note]
inline constexpr uint32_t kOpHandlerAbiVersion = 1;

// Initializer record for an input slot, when that slot names a Graph::initializers
// entry. byte_len is the ONLY correct byte source for Q4/Q8 (dtype_size()==0), so
// params/weight-byte handlers MUST read it, not shape*dtype_size. [critique-fix]
struct InitRef {
  int64_t   elem_count = 0;     // TensorRef::elem_count() (>0 when known)
  uint64_t  byte_len   = 0;     // recorded byte_len (truth for quantized)
  ir::DType dtype      = ir::DType::Unknown;
};

// ---------------------------------------------------------------------------
// OpContext — READ-ONLY node view. Every accessor is bounds-checked and returns a
// "not available" sentinel (nullptr / nullopt / Unknown / default) instead of
// throwing, so a handler over hostile/partial IR never crashes (inv 4/5). A NULL/
// absent/empty shape means "unresolved" -> honest-unknown (inv 6), exactly as
// compute_flops returns early when get_input_value()/get_output_value() are null
// or elem_count_from_shape()==0. Borrows (model,graph,node); copies nothing.
// ---------------------------------------------------------------------------
class OpContext {
 public:
  // --- Identity -----------------------------------------------------------
  std::string_view op_type() const;          // NORMALIZED: lowercased last dot-
                                             // segment (== CostModel.cpp norm_op).
                                             // This is the exact-tier registry key.
  std::string_view op_raw() const;           // original ("com.microsoft.QLinearConv")
  std::string_view domain() const;           // "com.microsoft" or "" if none
  OpCategory default_category() const;       // categorize_op(op_raw) — the color
                                             // tier key + the generic-formula branch
                                             // selector (Norm/Activation/Elementwise
                                             // -> |O|, Reduce -> |in0|, etc.).

  // --- Slot counts / shapes / dtypes -------------------------------------
  uint32_t input_count() const;
  uint32_t output_count() const;
  const Shape* input_shape(uint32_t slot) const;   // nullptr if slot/edge/value OOR
  const Shape* output_shape(uint32_t slot) const;  // (== get_input/output_value)
  ir::DType input_dtype(uint32_t slot) const;      // Unknown if slot absent
  ir::DType output_dtype(uint32_t slot) const;

  // --- Initializer (weight) membership + record --------------------------
  bool input_is_initializer(uint32_t slot) const;              // build_initializer_index
  std::optional<InitRef> input_initializer(uint32_t slot) const;  // elem_count/byte_len/dtype

  // --- Attributes (mirror CostModel.cpp's three helpers EXACTLY) ----------
  bool has_attr(std::string_view name) const;      // present AND of a known kind
  int64_t attr_int(std::string_view name, int64_t default_val) const;  // == get_int_attr
  std::string_view attr_string(std::string_view name,
                               std::string_view default_val) const;    // == get_string_attr
  const std::vector<int64_t>* attr_ints(std::string_view name) const;  // nullptr if absent/wrong-kind
  std::optional<double> attr_float(std::string_view name) const;

  // --- Constant integer INPUT (shape/index/axes tensors) — the ONE bytes-
  // adjacent accessor, reconciling with ShapeInferenceExt's constant-driven
  // rules (Reshape/Slice/Gather/Expand/Tile/Resize). Host reads it under EXACTLY
  // ShapeInferenceExt::read_shape_initializer discipline:
  //   raw_data initializer only (file_offset!=UINT64_MAX, not external);
  //   dtype I32/I64 only; elem_count in (0,64] (can never span a weight);
  //   does NOT call ByteReader::mark_payload_read (<=64-elem shape tensor is
  //   METADATA). Returns nullptr when ANY condition fails, OR when mmap_base_ is
  //   null (the COST driver leaves it null, so flops() handlers deterministically
  //   get honest-unknown; only the SHAPE driver, called with an mmap, resolves
  //   const inputs). [critique-fix: mmap state is part of the frozen struct]
  const std::vector<int64_t>* input_const_ints(uint32_t slot) const;

 private:
  const ir::Model* model_ = nullptr;
  const ir::Graph* graph_ = nullptr;
  const ir::Node*  node_  = nullptr;
  const uint8_t*   mmap_base_ = nullptr;   // set only by the shape-inference driver
  uint64_t         mmap_size_ = 0;
  friend class Registry;
};

// ---------------------------------------------------------------------------
// Result types
// ---------------------------------------------------------------------------
struct FlopResult {                 // honest-unknown by default (==NodeCost{0,false})
  uint64_t flops = 0;
  bool     known = false;
  static FlopResult unknown() { return {}; }
  static FlopResult of(uint64_t f) { return {f, true}; }   // MAC=2 is caller's job
};

struct ColorResult {                // default = use category color (view palette)
  bool  overridden = false;
  Rgba8 rgba;
  static ColorResult use_category() { return {}; }
  static ColorResult explicit_color(Rgba8 c) { return {true, c}; }
};

// Per-output-slot propagation, reconciled with infer_shapes_ext's MUTATE-IN-PLACE,
// best-effort, honest-unknown contract. The registry ADAPTER applies each entry
// with the SAME set_shape/carry_dtype semantics: write shape ONLY if the value's
// shape is currently empty; carry dtype ONLY if currently Unknown -> a handler can
// never clobber an already-resolved value, two handlers can't fight. A dim of -1
// is a legal dynamic dim, propagated as-is.
struct ShapeResult {
  struct Out {
    uint32_t  slot  = 0;
    Shape     shape;                       // empty => don't set shape for this slot
    ir::DType dtype = ir::DType::Unknown;  // Unknown => don't carry dtype
  };
  std::vector<Out> outputs;                // empty => handler resolved nothing
  static ShapeResult none() { return {}; }
};

// ---------------------------------------------------------------------------
// OpHandler — implemented by BuiltinOpHandler, DeclarativeOpHandler, WasmOpHandler.
// ---------------------------------------------------------------------------
class OpHandler {
 public:
  virtual ~OpHandler() = default;
  virtual OpCategory  category(const OpContext& ctx) const = 0;      // required
  virtual ColorResult color(const OpContext&) const { return ColorResult::use_category(); }
  virtual FlopResult  flops(const OpContext&) const { return FlopResult::unknown(); }
  virtual ShapeResult infer_shape(const OpContext&) const { return ShapeResult::none(); }
  virtual uint32_t    api_version() const = 0;   // == kOpHandlerAbiVersion or refused
};

}  // namespace netvis::plugin
```

### 2.2 `engine/plugin/ParserPlugin.h`

```cpp
// engine/plugin/ParserPlugin.h — FROZEN parser-extension ABI (v0.6.0).
//
// A ParserPlugin turns one whole file into an ir::Model, run ONCE on a worker
// thread. Built-in parsers (onnx/tflite/.../coreml) register as ParserPlugins on
// the SAME path a WASM parser would (dogfood). The WASM backend supplies an
// ADAPTER implementing this interface and marshalling across wasm3; no C++ object
// crosses the sandbox. Signature MIRRORS the real, precedent-frozen parser form
// `Result<ir::Model> parse(const MappedFile&, ProgressSink&)` — there is no
// HostFileView/ModelBuilder/ParseResult in the codebase; inventing them would fork
// every existing parser. [plan-correction]
//
// ZERO-PAYLOAD OBLIGATION (frozen contract, tested per plugin): the returned Model
// carries only TensorRef{offset,len,...} — the IR has NO decoded-payload field, so
// a plugin structurally cannot inject weight buffers into the view. A WASM parser's
// host_read_range is confined to declared-structure ranges and REJECTED (or marks
// the payload counter) on any read overlapping a recorded tensor range; the
// payload_read_counter()==0 test remains meaningful only because the HOST, not the
// plugin, owns that marking. [critique-fix, §Part 5]
#pragma once

#include <cstdint>
#include <string_view>

#include "core/JobSystem.h"    // ProgressSink
#include "core/MappedFile.h"   // MappedFile
#include "core/Result.h"       // Result<T>, err(offset)
#include "ir/IR.h"             // ir::Model
#include "parsers/Parser.h"    // Format

namespace netvis::plugin {

inline constexpr uint32_t kParserPluginAbiVersion = 1;

class ParserPlugin {
 public:
  virtual ~ParserPlugin() = default;

  // The Format this plugin emits. A third-party format not in the APPEND-ONLY
  // Format enum returns Format::Unknown and self-labels via ir::Model::format_name
  // (a StringId); the enum is NEVER renumbered/extended per plugin. Built-ins
  // return their real Format. [plan-correction]
  virtual Format format() const = 0;
  virtual std::string_view display_name() const = 0;

  // CONTENT sniff over the WHOLE mapped file (real sniffers need it: EOCD at the
  // file end, HDF5 probes to offset 8192, SafeTensors u64-len+JSON). NOT a bounded
  // head buffer. [plan-correction]  Pure, bounded, no payload read.
  virtual bool can_parse(const MappedFile& file,
                         const std::string& ext_hint) const = 0;

  // Priority for the content-sniff phase (lower = sniffed earlier). Reproduces
  // detect_format's fixed order (see §Part 4.4). User plugins default BELOW
  // built-ins so a plugin cannot hijack a known format.
  virtual int priority() const = 0;

  // Parse whole file -> ir::Model. IDENTICAL signature to every existing parser.
  // Records offset+len only (inv 3); returns err(offset) on malformed input
  // (inv 5); never throws.
  virtual Result<ir::Model> parse(const MappedFile& file,
                                  ProgressSink& progress) const = 0;

  virtual uint32_t api_version() const = 0;
};

}  // namespace netvis::plugin
```

### 2.4 `engine/plugin/Registry.h`

```cpp
// engine/plugin/Registry.h — FROZEN registry spine (v0.6.0, Increment 1).
//
// Immutable-snapshot, lock-free reads. Reads come from BOTH the worker thread
// (CostModel + infer_shapes_ext run as jobs) and the UI thread (per-frame color)
// concurrently, so the active table is a shared_ptr<const RegistryTable> swapped
// atomically on the rare user-driven reload (plugin enable/disable). Handlers are
// owned by unique_ptr and looked up ONCE PER OP-TYPE (hoisted), never copied per
// node — no std::function-copy cost on the 3095-node hot path.
#pragma once

#include <memory>
#include <optional>
#include <string>
#include <string_view>
#include <unordered_map>
#include <vector>

#include "engine/plugin/OpHandler.h"
#include "engine/plugin/ParserPlugin.h"
#include "engine/plugin/PassPlugin.h"

namespace netvis::plugin {

// Backend tier = PRIMARY key of the total override order (§0.5). Higher wins.
enum class Origin : uint8_t { Builtin = 0, Declarative = 1, Wasm = 2 };

struct OpResolution {
  const OpHandler* handler = nullptr;   // never null for a resolved op (builtin is catch-all)
  Origin           origin  = Origin::Builtin;
  bool             overrides_builtin = false;  // a user handler shadows a builtin key
  std::string_view plugin_name;                // "" for builtin
};

// IMMUTABLE once built.
struct RegistryTable {
  // Exact-op tier: normalized-op key -> the winning user handler (WASM/Declarative
  // after tie-resolution). Built-in is the monolithic catch-all below.
  std::unordered_map<std::string, OpResolution> op_by_key;   // owns key strings
  const OpHandler* builtin_op = nullptr;   // answers EVERY op; honest-unknown lives here
  std::vector<const ParserPlugin*> parsers;                  // sorted by priority()
  std::unordered_map<std::string, const PassPlugin*> passes; // display_name -> pass
};

class Registry {
 public:
  static Registry& instance();                              // Meyers singleton
  std::shared_ptr<const RegistryTable> snapshot() const;    // lock-free load

  // Two-tier op resolution (§Part 3.3): exact normalized-op key first, else the
  // builtin catch-all (which itself category-dispatches + honest-unknowns).
  OpResolution resolve_op(std::string_view op_norm) const;

  // Build an OpContext for a node. `mmap_base` non-null ONLY from the shape driver.
  OpContext make_context(const ir::Model&, const ir::Graph&, const ir::Node&,
                         std::string_view op_norm,
                         const uint8_t* mmap_base = nullptr,
                         uint64_t mmap_size = 0) const;

  // Content-sniff phase then a SEPARATE ext-hint fallback (§Part 4.4).
  Format detect_format(const MappedFile&, const std::string& ext_hint) const;
  const ParserPlugin* parser_for_format(Format) const;

  // Registration (used by built-in self-registration + plugin loaders). Refuses on
  // api_version mismatch: drops the handler, logs, never half-registers.
  void register_op_handler(std::string norm_key, std::string domain,
                           std::unique_ptr<OpHandler>, Origin, bool override_flag,
                           std::string plugin_name);
  void register_parser(std::unique_ptr<ParserPlugin>);
  void register_pass(std::unique_ptr<PassPlugin>);

  void reload(std::shared_ptr<const RegistryTable> next);   // atomic swap

 private:
  std::shared_ptr<const RegistryTable> table_;
};

}  // namespace netvis::plugin
```

## Part 3 — OpContext coverage proof (every built-in handler → exact OpContext calls)

`|O|` = `plugin::elem_count(*ctx.output_shape(0))`. All arithmetic via `safe_mul`/`safe_add`. **Every ported handler MUST reproduce the exact null/rank/positivity guards of its `CostModel.cpp` branch** — the table shows arithmetic only; guard fidelity is gated by the per-node FLOP-equality diff in Part 6. **[critique-fix: guards are load-bearing]**

| Built-in (CostModel.cpp) | Need | Exact OpContext expression (+ guards) | ✓ |
|---|---|---|---|

### 3.3 Two-tier resolution + hoist (fallback ladder) **[critique-fix: ambiguity resolved]**

`resolve_op(op_norm)`:
1. **Exact tier** — `op_by_key.find(op_norm)`. Present ⇒ return the user handler (already the tier winner per §0.5). `overrides_builtin` set iff the built-in also knows this key.
2. **Built-in catch-all** — `{builtin_op, Origin::Builtin, false, ""}`. The monolithic `BuiltinOpHandler` internally does the **category tier** (its `flops()` delegates to `compute_flops`, whose category-generic branches key off `categorize_op`) and returns honest-unknown for genuinely unknown ops (`flops_known=false`, `category==Other`). **Honest-unknown lives here.**

- **User handlers register ONLY under exact normalized-op keys** (a plugin cannot register a category-wide handler). Category-generic behavior remains the built-in's job. `categorize_op(raw)` drives the color/category tier; `norm_op(raw)` drives the exact tier — matching CostModel.cpp's two normalizations.
- **Hoist:** `resolve_op` is called **once per distinct op-type** per pass, cached in a stack-local `StringId → OpResolution` memo (Part 6), never per node.

### 5.1 Runtime: wasm3, vendored source-only, behind a runtime-agnostic boundary

### 5.2 Capability host API — enforcement by the exposed set

The zero-payload thesis is a property of the **output** import set: `ir::Model` has **no field** that can hold tensor contents (`TensorRef`={name,dtype,shape,file_offset,byte_len,external_path}; no `bytes` on any IR type), so **no host import accepts a content buffer**. Exhaustive input set `{host_file_len, host_read_range}`; exhaustive Model-mutating set `{host_intern, host_add_value, host_add_node, host_add_attr_*, host_record_tensor, host_set_metadata, host_set_model_info, host_set_error, host_progress}`. None accepts anything but StringId/int64/double/DType/dims/offset/len/index.

**Zero-payload enforcement moved into the host read boundary [critique-fix]:** because `payload_read_counter()` is honor-system (only `mark_payload_read` increments; `bytes()`/`raw()` do not), a naive `host_read_range` over `raw()` could read weight bytes with the counter still 0. Frozen rule:
- The host tracks the set of tensor ranges recorded via `host_record_tensor`.
- `host_read_range(off,len)` is **rejected** if `[off,off+len)` overlaps any recorded tensor range (returns nonzero, no-op). A parser reads structure *before* recording tensors; any attempt to re-read a recorded payload region fails.
- Additionally, the host itself calls `ByteReader::mark_payload_read()` if it ever must permit such a read, so the counter is the HOST's accounting, not the plugin's. The per-plugin test asserts `payload_read_counter()==0` **and** that every produced `TensorRef` is offset+len.
- `host_read_range` bounds: `off+len<=file_len` (overflow-guarded), `len<=kReadChunkCap` (1 MiB), cumulative `<=kStructBudget` where **`kStructBudget = max(1 MiB, min(16 MiB, 0.05·file_len))`** — the floor fixes the small-file starvation (a 2 KB toy model gets 1 MiB, not 100 bytes). **[critique-fix]**

NOT exposed: any tensor-content read, filesystem, network, process/env/clock/RNG, WASI, host-heap alloc, `AttrValue::Kind::Tensor` byte channel.

**Op-handler string/list attr channel [critique-fix]:** §2c's "no imports for op handlers" is dropped. A WASM op handler gets a **read-only `host_str(str_id,dst,cap)->len`** import AND the packed op-context record inlines every string attr's UTF-8 bytes and every ints/floats-list attr as `(offset,len)` blobs. Without this a WASM handler could not reproduce Einsum's `equation` or Pool's `kernel_shape`; the exact wire layout is specified in `docs/plugin-abi.md`. The host **clamps the returned category to `[0, Other]`** before any `1u<<cat` mask op or palette index (a hostile `category=200` would otherwise be shift-UB / mask corruption). **[critique-fix]** The op-handler result path returns shapes+dtype+flops+category+rgba only — it accepts **no file offsets**, so it cannot smuggle an out-of-range `file_offset` into a `ValueInfo`. **[critique-fix]**

### 5.3 Sandbox: fuel + memory + watchdog

- **Memory:** `max_pages` (default 256 = 16 MiB); reject modules declaring `max` above it AND trap on `memory.grow` past it.
- **Fuel:** gas counter decremented at back-edges (`op_Loop`/`op_Branch_r`/`op_Call`/`op_CallIndirect`) via the vendored patch; traps at 0. **Backstop [critique-fix]:** because a straight-line or in-host-import stall does not hit a back-edge, the interpreter loop ALSO polls the atomic abort flag on a periodic timer (not only at the gas site), and the `.wasm` code-section size is bounded — so the watchdog (`watchdog_ms`, default 2000) genuinely fires within its window.
- **Abort path:** any trap ⇒ plugin disabled for the session (persisted disabled-with-reason to `view_prefs.json`), app survives (trap unwinds into the worker thread; `ir::Model` mutated only through transactional single-append imports ⇒ consistent partial model), returns partial+labelled (`netvis.partial`) or `Error(offset)` via the generation-checked completion queue. Fresh `WasmInstance` per call.

### 5.4 Parser hook + SDK

`WasmParserPlugin` implements `ParserPlugin` with `priority()` **below all built-ins**; it is consulted only in `parse_model`'s `Format::Unknown` branch — iterate registered WASM `can_parse` over fresh instances, first positive claim runs `parse`, else fall through to today's Unknown error. Built-in formats always short-circuit first (tie → built-in). **[critique-fix: explicit hook point]**

SDK ships `plugins/sdk/netvis_plugin.h` (C, freestanding: import decls, export macros, `DType` wire enum = `ir::DType` numeric order, bump allocator, `kAbiVersion`), examples `toy_parser/` (magic `NVTOY1`, records offset+len only, golden test asserts counter==0) and `attn_share_pass/`, and `docs/plugin-abi.md`. Toolchains: `clang --target=wasm32 -nostdlib -Wl,--no-entry`, Rust `wasm32-unknown-unknown`, Zig `wasm32-freestanding`. One `.wasm` runs on all three OSes.

### Q1 — Manifest format: **JSONC via the already-linked `nlohmann/json`. Not TOML.** (§0.1)

Adds `std::optional<OpCategory> category_from_name(std::string_view)` to `OpCategory.h` (trivial inverse of `category_name`), so `category` typos are rejected, never silently `Other`.

### Q2 — DSL surface: fixed operator set, `vars` DAG, signed-checked int64, honest-unknown propagation

- **Value model:** `Val{int64_t v; bool known;}`. Integer only. Arithmetic in **signed checked int64** (`checked_add/sub/mul/div/mod_i64` from `SafeMath.h`); **any overflow OR any negative value feeding a FLOP/element-count ⇒ unknown**, never the unsigned-`safe_mul` wrap-to-huge that would fabricate `flops_known=true`. `div`/`mod` guard both `b==0` and `INT64_MIN/-1`. **[critique-fix ×2]**
- **Operators:** `+ - * / %`, comparisons, `&& || !` (C short-circuit), ternary `?:`, `min/max`, `prod(...)`.
- **Terms:** integer literals; `inK.shape[i]`/`outK.shape[i]` (negative `i` from end); `inK.rank`; `O` (output-0 elem count); `nin`/`nout`; **`attr("k")`** (unknown if absent) and **`attr("k", default)`** (default if absent/wrong-kind — mirrors `get_int_attr`, so a declarative Gemm with omitted `transA` resolves KNOWN, matching the built-in) **[critique-fix]**; `attr("k", i)` (ints element); **`sattr("k")=="literal"`** string-equality predicate + **`attr_enum("k", {"bidirectional":2, ...})`** (so RNN `direction` is expressible; a bare `sattr` value is usable only inside ternary predicates, keeping arithmetic int64) **[critique-fix]**; `ident` (a `vars` entry). **`prod(inK.shape[a:b])` slice-reduction** (bounds are DSL int exprs clamped to `[0, rank]`, `<1⇒unknown`) so rank-agnostic Conv/Pool kernel volume (`partial_shape_product(w,2,rank-2)`) is expressible **[critique-fix]**; and an explicit **`unknown`** literal (yields `{known=false}`) as the canonical honest-unknown guard (`w.shape[-1]%3==0 ? real : unknown`, so a guard failure is NOT a fabricated known-0). **[critique-fix]**
- **`vars`** form a cycle-checked DAG (bounded DFS); may reference other vars/operands/attrs/O/nin/nout. Memoized in dependency order per node.
- **Dynamic-dim propagation (§0.3):** unknown is contagious except through short-circuited `?:`/`&&`/`||`. `O` narrows a `uint64` elem count to int64 with a **saturating clamp to INT64_MAX** (never a wrapping cast). FLOPs: `known = val.known && val.v>=0`. Shape: unknown dim ⇒ `-1`, write if ≥1 dim resolves.
- **Scope of declarative override (§0.8):** metadata/shape-arithmetic ops only. String/loop-driven ops (Einsum, per-label logic) are WASM/built-in only; this is documented so `override:true` claims stay honest.
- **`O` ordering [critique-fix]:** `ShapeInferenceExt` (including plugin shape exprs) runs to fixpoint **before** `compute_flops`; `O` reads the post-inference output-0 shape; any `-1`/`0` dim ⇒ `O` unresolved ⇒ `flops_known=false`, excluded from totals.

`DeclarativeOpHandler` holds a `shared_ptr<const CompiledOp>` (AST compiled once at load), so lookup is a cheap handle and there is no per-node allocation.

**Safety limits (fatal-to-op, logged):** parse depth ≤ 64, tokens/expr ≤ 4096, expr source ≤ 16 KiB, vars/op ≤ 256, ops/file ≤ 4096, file ≤ 4 MiB (checked before parse), int literal must fit int64, `prod/min/max` args ≤ 256. Full fuzz list (depth/token/ternary bombs, unterminated strings, huge literals, div0, `INT64_MIN/-1`, OOB shape index, missing slot/attr, negative-flops, subtraction underflow, saturation, cycle reject, api_version mismatch → nothing registered, `payload_read_counter()==0` after `compute_cost` with declarative ops) is the Increment-2 acceptance suite.

### Q3 — Override precedence: explicit `override:true`, single total order, visible indicator (§0.5)

- A user handler colliding with a **built-in** key is **rejected unless `override:true`** (silent shadowing of a FLOP formula is forbidden — inv 6 spirit).
- **Total order [critique-fix]:** backend tier is the **primary** key (WASM > declarative > built-in); enumeration order (lexicographic absolute path) is the **tiebreaker within a tier**. So a WASM override always beats a declarative one regardless of filename; two declarative files defining the same op resolve by path order (later = shadowed). The comparator is a single deterministic function; the loser is shown "shadowed by X" in every case.
- **Indicator:** Properties/Cost panel shows `FLOPs from plugin "X" (overrides built-in)`; the Plugins panel lists each plugin's overridden built-ins. Computed at table-build time (zero hot-path cost).

### Q4 — Runtime: **wasm3 only for v0.6.0; runtime-agnostic host boundary** (§5.1)

## Part 7 — Discovery, trust, versioning, panel (#11)

### 7.1 Discovery
Two roots, fixed order: (1) `<exe-dir>/plugins/`, (2) a new **`plugin_dir()`** helper — a **config-tree** sibling of `layout_cache_dir()` **[plan-correction]** (the plan's literal "sibling of `layout_cache_dir()`" would put plugins under the wipeable *cache* tree): Linux `$XDG_CONFIG_HOME/netvis/plugins` else `~/.config/netvis/plugins`; macOS `~/Library/Application Support/NetVis/plugins`; Windows `%LOCALAPPDATA%\NetVis\plugins`. Each plugin is a directory with one `plugin.json` (+ referenced `*.wasm`). Enumeration: all manifests from root 1 sorted by absolute path, then root 2 — fully deterministic. Name collisions across roots: first loads, second marked **shadowed**.

### 7.2 Trust
| Backend | Default | Gate | Badge |
|---|---|---|---|
| Declarative (JSON+DSL) | **Enabled** | none — safe by construction (no side-effecting/byte term, arithmetic checked, cycle-free) | green "safe" |
| WASM (wasm3) | **Disabled** | explicit per-plugin enable + one-time capabilities confirm dialog | amber "sandboxed code" |
| Native `.so`/`.dll` | **Never** | — | not discovered |

### 7.3 Versioning
`constexpr uint32_t kOpHandlerAbiVersion / kParserPluginAbiVersion / kPassPluginAbiVersion = 1` — three independent constants so a shape-result change doesn't force a parser re-release. **Bump only on a breaking change** to the corresponding types; a new OpContext accessor or a new defaulted virtual is additive and does NOT bump (which is why `color/flops/infer_shape` all have defaults). **Exact `==` match; mismatch ⇒ plugin refused, never half-loaded**, panel shows `requires API vN, host is vM`. These constants gate DSL manifest `api_version` and the WASM module's `netvis_abi_version()` — **not** a C++ vtable ABI (single binary, all compiled together). **[invariant-note]**

### 7.4 Plugins panel + persistence
New `src/view/PluginsPanel.cpp` (confirm `src/view` is a target-sources dir; if it is not globbed like `parsers`/`engine`, add the TU explicitly — the "zero CMake edits" claim only holds for `src/parsers/*` and `src/engine/*`). Rows: name·author, type badge, `api_version` cell (green/red), status (`Loaded`/`Disabled`/`Rejected:<reason>`/`Shadowed by X`), enable checkbox (greyed for Rejected; WASM off→on shows the confirm dialog), expandable detail (capabilities, overridden built-ins, path, error).

Persistence: one new **per-key-guarded** `"plugins"` object `{name: bool}` in `view_prefs.json`, following `save_prefs`/`load_prefs` exactly. Missing key ⇒ safe default (declarative enabled, WASM disabled). Never prune unseen entries. **Do NOT bump `LayoutCache::kVersion`** (Part 6 #7).
