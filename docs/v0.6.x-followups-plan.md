> **Status: design / contracts freezing.** v0.6.0 (plugins) is **merged + released**
> (tag `v0.6.0`): registry spine (#8), declarative DSL (#9), WASM sandbox + `PassPlugin`
> host API (#10 partial), plugins panel + trust/versioning (#11 partial). This document
> is the authoritative spec for the four v0.6.x follow-up increments that finish
> **#10** (WASM OpHandler + ParserPlugin adapters + SDK) and **#11** (per-plugin
> enable/disable persistence + WASM trust gate). It extends the merged code under
> `src/engine/plugin/` — it does not reinvent it. Every design decision that folds an
> adversarial-critique blocking-issue fix is marked **[critique-fix]**.
>
> Structured after `docs/v0.6.0-design.md`. The four frozen ABI headers
> (`OpHandler.h`, `ParserPlugin.h`, `PassPlugin.h`, `Registry.h`) are **not** re-frozen
> here; this increment adds only WASM *host imports* (new wire ABI) and *view-side*
> code. The freeze-contract method applies to the new host imports: they are frozen
> (SDK header + `g++ -fsyntax-only` + wasm3 sig-string audit) **before** any adapter
> lands against them.

## Non-negotiable invariants (every increment upholds all six)

1. **Single self-contained binary** — wasm3 already vendored (`FetchContent_Populate`, WASI TUs excluded); no new third-party dependency, no dlopen.
2. **Cross-platform** — every new wasm3-including TU repeats the `#pragma warning(push/pop, disable:4200)` + `-Wpedantic` guard block (`WasmHost.cpp:15-30`) and the `C4100` guard (`WasmHost.cpp:37`) around `m3ApiRawFunction`. **NO `std::atomic<shared_ptr>`** (libc++ lacks the specialization — the Registry mutex pattern, `Registry.h:104-109`, is reused; any new shared state is mutex-guarded).
3. **Zero tensor-payload reads** — no host import returns a decoded weight buffer. For the parser facet this is **not** a property of the import set alone (there is a byte-read import); it is a **bounded-read + host-owned-marking** property — see Increment B §0.1. **[critique-fix]**
4. **Hostile-input-safe** — every WASM call is fuel- and memory-capped; every guest-supplied offset/count is validated in checked `uint64` against `[NV_NULL_GUARD_OFF, memsize]` before dereference; category is clamped before any `1u<<cat` / palette index.
5. **Never throw / crash** — trap / fuel-exhaustion / load-error → honest-unknown facet or `err(offset)` `Result`; the app always survives.
6. **Honest-unknown** — abstain status, unresolved shape (`rank -1`), trapped facet all funnel to `FlopResult::unknown()` / `ShapeResult::none()` / `default_category()`; never a fabricated value.

## Part 0 — Cross-increment contradictions, resolved once

Load-bearing decisions the four facet designs disagreed on. Settled here, referenced everywhere.

| # | Contradiction | **Frozen answer** |
|---|---|---|
| 0.1 | WASM parser zero-payload: "property of the import set" vs the reality that `host_read_range` **is** a byte-read import | **Bounded-read + host-owned marking, NOT import-set purity.** `host_read_range` is confined to a small up-front sniff window (head + tail), marks every returned byte via a host-side `ByteReader`, and is barred from any recorded-tensor range on **every** call. The witness becomes "structural reads ≤ declared bound" — `payload_read_counter()` reflects *only* host-marked structural reads. The design honestly drops the "counter==0" claim for the WASM parser facet. **[critique-fix]** |
| 0.2 | WASM handler thread affinity: live from `resolve_category` (render thread) vs worker-pass memo | **NEVER invoke a WASM handler from `resolve_category` (render thread).** WASM category/color/flops/shape are resolved **once per op-type on the worker thread** during the cost/shape pass and memoized into a per-op-type style/cost cache the UI reads. The draw path never enters the sandbox. **[critique-fix]** |
| 0.3 | WasmEngine concurrency: "one module per worker" assumed single-threaded vs the real UI+worker split | **One process-wide mutex serializes ALL `WasmEngine` load/instantiate/call access.** wasm3's `IM3Environment` is shared mutable state (`m3_ParseModule` mutates `env->funcTypes`); the singleton gains `g_wasm_engine_mutex` (a plain `std::mutex`, portable — inv 2). No `std::atomic<shared_ptr>`. **[critique-fix]** |
| 0.4 | Enable-gate: runtime flag vs table membership | **Enforced by TABLE MEMBERSHIP, not a runtime flag.** A disabled WASM plugin is **structurally absent** from `op_by_key`/`parsers`/`passes`; discovery records it (for the panel) but registers only *enabled* plugins. Toggling triggers `Registry::reset_to_builtins()` + re-discovery. `resolve_op` / `Detect.cpp` can never reach a disabled plugin — so untrusted `can_parse` (which *executes* code) never runs for a disabled parser. **[critique-fix]** |
| 0.5 | SDK 64-bit wire types | **Every offset/length/i64-return uses `int64_t`/`uint64_t` in the SDK header, never C `long`** (which is 32-bit on wasm32). Each import decl is audited against the wasm3 sig-char table (`I`=i64, `i`=i32, `F`=f64, `f`=f32, `*`=i32-offset). **[critique-fix]** |
| 0.6 | Marshalling bounds on mutation imports | **Every guest-supplied count is capped in the SDK header AND host-validated** in checked `uint64` before `m3ApiCheckMem`: `NV_MAX_RANK` (8), `NV_MAX_NODE_IO`, `NV_MAX_ATTR_LEN`, `NV_MAX_INTERN_LEN`, `NV_ERR_MSG_MAX` (1 KiB), plus host-side `ParseLimits` on total graphs/nodes/values/tensors. Reject out-of-range (`-1` / honest-unknown), never abort. **[critique-fix]** |

## Increment A — WASM `OpHandler` adapter (closes #10, op facet)

### Scope

`WasmOpHandler` is the fourth `OpHandler` implementor alongside `BuiltinOpHandler` (`Registry.cpp`), `DeclarativeOpHandler` (`Manifest.cpp`), answering `category/color/flops/infer_shape` over a read-only `OpContext`, backed by a sandboxed WASM module. **This is the documented declarative-escalation SLOW PATH** (roadmap Part B): the DSL is preferred; WASM op handlers are for FLOP/shape logic exceeding the checked-int64 expression DSL. Per-node WASM invocation is a footgun, mitigated below and honestly badged in-product.

### A.1 Thread model — the crux [critique-fix]

**A WASM op handler is NEVER invoked from `resolve_category` on the render thread** (`Registry.cpp:356`, called per visible node per frame from `GraphCanvas`/`GraphNav`/`PropertiesPanel`). Doing so would (a) race the singleton `IM3Environment` against a concurrent worker-thread `infer_shape`, and (b) re-parse+link a full module every frame with a permanently-tripping guard. **[critique-fix ×2]**

Instead:
- WASM category/color/flops are resolved **once per distinct op-type on the worker thread** during `compute_graph_costs` (which already memoizes the resolved handler per op-type, `CostModel.cpp`), and cached into the per-op-type cost/style state the UI already reads.
- `resolve_category` gains a fast path: if the resolved handler's origin is `Wasm`, it returns the **cached** category scalar (populated by the last worker pass) rather than entering the sandbox; before the first pass, or for an op-type not yet cached, it returns `ctx.default_category()` (honest, deterministic).
- All `WasmEngine::load`/`call_i32` access is serialized by a process-wide `std::mutex` in `WasmEngine::instance()` (§0.3). **[critique-fix]**

### A.2 Host-import set to READ an OpContext (module `"netvis_op"`) — FROZEN

New `m3ApiRawFunction`s in `WasmOpHandler.cpp`, linked by `link_op_capabilities(IM3Module, OpHostCtx*)`, mirroring `link_capabilities` (`WasmHost.cpp:106`). Threaded userdata:

```cpp
struct OpHostCtx {
  const OpContext* ctx = nullptr;      // borrows; valid only during one call
  // captured results (host reads after the entry call returns):
  bool have_category=false; int32_t raw_category=0;
  bool have_flops=false;    uint64_t flops=0; bool flops_known=false;
  bool have_color=false;    uint32_t rgba=0;
  ShapeResult shape;
};
```

**Marshalling rule:** scalars return by value; variable-length data is written into a guest-supplied `(dst_ptr, dst_cap)` and the host returns the count actually written, every write bounds-checked via `m3ApiCheckMem` (as `host_emit_metric`, `WasmHost.cpp:93`). No import returns a weight buffer (inv 3 — the only bytes-adjacent reads are ≤64-elem shape metadata + initializer counts/lengths, inheriting `OpContext`'s existing no-payload-bump discipline, `OpHandler.h:89-100`).

Read imports (`i(...)`/`I(...)`/`f(...)`):
- `i32 op_input_count()`, `i32 op_output_count()`, `i32 op_default_category()`
- `i32 op_type(i32 dst, i32 cap)` → bytes written of normalized `op_type()`
- `i32 op_input_rank(i32 slot)` / `op_output_rank(i32 slot)` → `-1` if unresolved (honest-unknown)
- `i32 op_input_dims(i32 slot, i32 dst, i32 cap)` / `op_output_dims(...)` → `i64[]` dims, count written; `-1` OOR
- `i32 op_input_dtype(i32 slot)` / `op_output_dtype(i32 slot)` → `ir::DType` (`Unknown`=15)
- `i32 op_input_is_initializer(i32 slot)`; `I64 op_input_init_elem_count(i32 slot)`; `I64 op_input_init_byte_len(i32 slot)` (Q4/Q8 truth); `i32 op_input_init_dtype(i32 slot)`
- `i32 op_has_attr(i32 name_ptr, i32 name_len)` (`name_len<256`); `I64 op_attr_int(i32,i32,I64 def)`; `i32 op_attr_string(i32,i32,i32 dst,i32 cap)`; `i32 op_attr_ints(i32,i32,i32 dst,i32 cap)`; `f64 op_attr_float(i32,i32,i32 known_ptr)`
- `i32 op_input_const_ints(i32 slot, i32 dst, i32 cap)` — the ONE guarded shape-tensor accessor; `-1` when `mmap_base_==nullptr` (cost driver always gets `-1` — deterministic honest-unknown, `OpHandler.h:96`).

Result out-imports (guest pushes results, host records into `OpHostCtx`):
- `void op_set_category(i32 cat)` — records RAW (clamped host-side, §A.3)
- `void op_set_flops(I64 value, i32 known)` — `flops = value<0?0:value; known = (known && value>=0)`
- `void op_set_output_shape(i32 slot, i32 dims_ptr, i32 rank, i32 dtype)`
- `void op_set_color(i32 rgba)`

**Bounds on `op_set_output_shape` [critique-fix]:** `rank` is validated `>= 0` first, then clamped `r = min(rank, NV_MAX_RANK=8)`, and the bounds check uses the **clamped** byte count: `m3ApiCheckMem(dims_ptr, r*8)` — never the raw guest `rank*8` (a negative/huge `rank` would negate/overflow the span and make the check meaningless). `dtype` clamped to a valid `ir::DType` else `Unknown`; a dim `<1 && != -1` coerced to `-1` (dynamic).

Exports (frozen `()->i32`, validated by `call_i32`, `WasmRuntime.cpp:111`): `netvis_op_abi_version` (== `kOpHandlerAbiVersion`, checked at load, refuse on mismatch — `Registry.cpp:312` pattern), `netvis_op_category`, `netvis_op_flops`, `netvis_op_infer_shape`, `netvis_op_color` (optional). The `i32` return is a **status** (0=answered, nonzero=abstain→honest-unknown); results flow through out-imports so the frozen zero-arg/i32-ret contract holds. **A missing export is surfaced as a load-time diagnostic in the Plugins panel**, not a silent per-call fallback. **[critique-fix]**

### A.3 THE CLAMP (mandatory) [critique-fix]

A hostile guest can `op_set_category(9999)`. The view does `1u << static_cast<uint32_t>(c)` (`GraphNav.cpp:175`) and indexes `kPalette[...]` (`App.cpp:538`). The host clamps once, at the marshal boundary, so no invalid `OpCategory` ever enters the IR/view:

```cpp
static OpCategory clamp_category(int32_t raw) {
  if (raw < 0 || raw > static_cast<int32_t>(OpCategory::Other)) return OpCategory::Other;
  return static_cast<OpCategory>(raw);
}
```

`OpCategory::Other` is the guaranteed-last enumerator (`OpCategory.h:16-20`); a clamped category is `< 32`, so `1u<<cat` is well-defined and `kPalette` indexing is in-range. `color()` forces `a=255`; `infer_shape()` returns `hc.shape`, applied by the driver with write-only-if-empty / carry-dtype-only-if-Unknown semantics (`OpHandler.h:135`) so a hostile shape can never clobber a resolved value.

### A.4 Fuel / mem caps + per-pass perf guard

```cpp
SandboxLimits kOpLimits{ .max_memory_pages=64 /*4 MiB*/, .max_steps=2'000'000 /*~100x below pass default*/ };
```

Trap/fuel/load-error on any facet → honest-unknown (`FlopResult::unknown()`, `ShapeResult::none()`, `category()→default_category()`); app survives (as `WasmPassPlugin::run`, `WasmHost.cpp:140`). `WasmOpHandler` holds the immutable `image_` bytes and constructs a fresh `WasmModule` per invocation under the engine mutex (no live module shared across threads).

**Perf guard (footgun mitigation):** guard state lives **outside** the const handler (respecting `RegistryTable` immutability) as a **worker-thread-local budget map keyed by handler pointer**, reset by the cost/shape driver at loop entry. After `kMaxInvocations` (50'000) or cumulative `> kBudgetMs` (250 ms) within one pass, the handler short-circuits to honest-unknown and the pass records a diagnostic surfaced in the Plugins panel ("wasm op '<name>' disabled: per-pass budget exceeded"). Because the sandbox is never entered from the draw path (§A.1), the guard has exactly one caller class (the worker drivers) and a well-defined reset — the permanently-tripping-`thread_local` failure mode is eliminated. **[critique-fix]**

### A.5 Registration

Sidecar `plugin.json` next to the `.wasm` (reusing `plugin_dir()`): `{ "api_version":1, "name":"...", "wasm":"handler.wasm", "ops":[{"name":"MyCustomConv","override":true}] }`. Loader `load_wasm_op_plugin`: verify `api_version`, dry-load + call `netvis_op_abi_version` under fuel, then per op `register_op_handler(normalize_op_key(name), domain, make_unique<WasmOpHandler>(shared_image, entry_names), Origin::Wasm, /*override=*/true, plugin_name)`. `Origin::Wasm` (2) outranks `Declarative` (1) and `Builtin` (0); `override:true` is required to shadow (`Registry.cpp:321-323`). Registration only for **enabled** plugins (§0.4 / Increment C).

## Increment B — WASM `ParserPlugin` + host parser capability API (closes #10, parser facet)

### Scope

A WASM parser turns a whole file into an `ir::Model`. **The host owns the `ir::Model`**; the plugin issues append-only *commands* over a finite import set and can never hold a host pointer. This is the highest-risk increment (it touches the zero-payload thesis surface), so §0.1's bounded-read + host-marking mechanism is load-bearing and every mutation import is bounded.

### B.1 Zero-payload mechanism — REDESIGNED [critique-fix]

The earlier "record-before-read overlap guard" is **rejected as unsound**: it is order-dependent (read-before-record bypasses it) and voluntary (the plugin chooses what to declare as a tensor), and `host_read_range` bypasses `ByteReader` so `payload_read_counter()` stays 0 while payload leaks. **[critique-fix]** The frozen mechanism instead:

1. **`host_read_range` is confined to a small, up-front sniff window** — head `NV_SNIFF_HEAD` (4 KiB) + tail `NV_SNIFF_TAIL` (4 KiB) of the file, sufficient for magic/format-directory detection (EOCD-style tails included). Reads outside the window are rejected (`-1`).
2. **The host reads through a real `ByteReader`** (not a raw `memcpy` from `MappedFile::data()`), so every returned byte is **marked** and bounded; the WASM-parser zero-payload test asserts the counter reflects **only** the bounded structural window, never weights. §0.1.
3. **No file-derived bytes reach a rendered `Model` field via an arbitrary-memory sink.** `host_intern`-from-arbitrary-wasm-memory is removed as an exfil channel: names / metadata / model-info are supplied as **`(file_offset, len)` ranges the HOST reads** (through the marked, window-bounded `ByteReader`), so the same overlap + window checks apply. At parse-end the host **re-validates** that no rendered string's source range overlaps any recorded tensor range and rejects the whole `Model` if it does. **[critique-fix]**
4. **Tensors are declared by `(file_offset, byte_len)` only** — `TensorRef` carries no bytes (`ir/IR.h`), so a tensor is structurally incapable of holding payload; `host_record_tensor` copies nothing.

`ParserPlugin.h`'s frozen contract comment is updated (a doc-only edit, not an ABI change) to state that for the WASM facet the zero-payload witness is the **bounded-structural-read counter**, not `counter==0`.

### B.2 Host-import set (module `"netvis"`) — FROZEN (all 64-bit via `int64_t`, §0.5)

New `m3ApiRawFunction`s in `WasmParser.cpp`, linked by `link_parser_capabilities(IM3Module, ParseHostCtx*)`. `ParseHostCtx{ const MappedFile* file; ByteReader reader; ir::Model* model; ParseLimits limits; std::vector<std::pair<uint64_t,uint64_t>> recorded_ranges; std::vector<StrSrc> rendered_string_srcs; std::string error; bool aborted; }`.

Read (window-bounded, marked):
- `I64 host_file_len()` — `"I()"`
- `i32 host_read_range(I64 off, i32 len, i32 dst)` — `"i(Iii)"` — copies `[off,off+len)` **only if inside the sniff window and not overlapping a recorded tensor range**, via the marked `ByteReader`; `len` capped at `NV_READ_CHUNK_CAP`; returns bytes copied or `-1`.

Model-mutating (host reads guest ranges; every count capped, §0.6):
- `i32 host_intern_range(I64 off, i32 len)` — `"i(Ii)"` — interns bytes the **host** reads from the marked window (records the source range for the parse-end re-validation); `len` capped `NV_MAX_INTERN_LEN`; returns `StringId.id`.
- `void host_set_model_info(i32 fmt_id, i32 prod_id, i32 ver_id)` — `"v(iii)"`
- `void host_set_metadata(i32 key_id, i32 val_id)` — `"v(ii)"` (capped `max_metadata`)
- `void host_set_error(i32 msg_ptr, i32 len)` — `"v(ii)"` — `len` capped `NV_ERR_MSG_MAX` (1 KiB), truncate + `m3ApiCheckMem`; closes the exfil-via-error channel. **[critique-fix]**
- `i32 host_begin_graph(i32 name_id)` — `"i(i)"` (capped `max_graphs`)
- `i32 host_add_value(i32 g, i32 name_id, i32 dtype, i32 dims_ptr, i32 rank)` — `"i(iiiii)"` — `rank` validated `[0, NV_MAX_RANK]`; `rank==0` skips the pointer check (scalar); `dtype` clamped to `Unknown`; clean per-call `-1` reject on `CheckMem` failure (never abort). **[critique-fix]**
- `i32 host_add_node(i32 g, i32 op_id, i32 name_id, i32 in_ptr, i32 nin, i32 out_ptr, i32 nout)` — `"i(iiiiiii)"` — `nin`/`nout` capped `NV_MAX_NODE_IO`; `0` skips the pointer check; out-of-range value indices dropped. **[critique-fix]**
- `void host_set_graph_io(i32 g, i32 in_ptr, i32 nin, i32 out_ptr, i32 nout)` — `"v(iiiii)"`
- `i32 host_add_attr_int(i32 g, i32 node, i32 name_id, I64 v)` — `"i(iiiI)"`; `_float` `"i(iiiF)"`; `_string` `"i(iiii)"`; `_ints(i32 g,i32 node,i32 name_id,i32 ptr,i32 count)` `"i(iiiii)"` — `count` capped `NV_MAX_ATTR_LEN`.
- `i32 host_record_tensor(i32 g, i32 name_id, I64 off, I64 len, i32 dtype, i32 dims_ptr, i32 rank)` — `"i(iiIIiii)"` — the sole weight declaration; `rank` capped; records `[off,off+len)` into `recorded_ranges`; copies zero bytes; `g<0` targets `Model::flat_tensors`.

**NOT exposed:** any tensor-content read, `host_intern` from arbitrary wasm memory, filesystem/network/clock/RNG/env, WASI, host-heap alloc, `AttrValue::Kind::Tensor` byte channel.

### B.3 `WasmParserPlugin` adapter

`WasmParserPlugin final : public ParserPlugin` in new `WasmParser.{h,cpp}`, `#else` no-op stubs (as `WasmHost.cpp:146`). `format()==Format::Unknown` (self-labels via `host_set_model_info`); `priority()` below all built-ins; `api_version()==kParserPluginAbiVersion`.
- `can_parse` — **loads and RUNS `netvis_can_parse`** in a fresh sandbox (tiny fuel, 1 page). Because this executes untrusted code, it runs **only for enabled plugins** (§0.4); a disabled parser is structurally absent from `parsers[]` and never sniffed.
- `parse` — builds `ParseHostCtx`, calls `netvis_parse` under `parser_run_limits` (fuel larger than the pass default but finite; memory MIN-clamped before load, `WasmRuntime.cpp:184`); on `Trap`/`FuelExhausted`/`LoadError` returns `err(...)` and **discards the partial Model** (honest); on `ret!=0`/`aborted` returns the (length-capped) error; on empty model returns honest-unknown. `Model::strings` growth is bounded by `NV_MAX_INTERN_LEN × max_intern_calls` regardless of trap timing.

### B.4 Detect hook

Built-in parsers are the `switch` in `parse_model` (`Detect.cpp`); WASM parsers hook **only** the `Format::Unknown` branch:

```cpp
case Format::Unknown:
  if (auto r = plugin::Registry::instance().try_unknown_parsers(file, ext_hint, progress)) return r;
  return err("unrecognized model file format", UINT64_MAX);
```

## Increment C — Plugin enable/disable persistence + WASM trust gate (closes #11)

### Scope

Persist per-plugin enable state in `view_prefs.json`; enforce the gate by **table membership** (§0.4) so a disabled WASM plugin's ops/parsers never reach the Registry and never execute. Declarative plugins default enabled (safe by construction); WASM defaults disabled behind a one-time confirm dialog.

### C.1 `view_prefs.json` schema addition

A `"plugins"` object alongside the existing flat keys, keyed by the **discovery subdirectory name** (stable/unique; the manifest `"name"` may be empty/duplicated so it is NOT the key):

```json
{ "dark_theme": true, "plugins": { "attn-ops": true, "my-wasm-pass": false } }
```

Absence of a key = safe default (declarative ENABLED, WASM DISABLED); entries for plugins not present this session are **never pruned**. Backing type in **core** (so it is headless-testable — `App` is view-only, not linked by tests):

```cpp
// engine/plugin/PluginPrefs.h  (uses <nlohmann/json_fwd.hpp> — an explicit build dep of the view+core targets)
enum class PluginKind : uint8_t { Declarative=0, Wasm=1 };
bool plugin_default_enabled(PluginKind k);   // Declarative => true, Wasm => false
class PluginEnableSet {
 public:
  bool effective(std::string_view id, PluginKind k) const;   // explicit else default
  void set(std::string id, bool on);
  bool has_explicit(std::string_view id) const;
  void load_json(const nlohmann::json& o);   // per-key bool guard; NEVER prunes
  nlohmann::json to_json() const;            // std::map => deterministic order
 private:
  std::map<std::string,bool> explicit_;
};
```

`App::save_prefs` adds `j["plugins"] = plugin_enabled_.to_json();`; `load_prefs` (inside the existing `try`) adds `if (j.contains("plugins") && j["plugins"].is_object()) plugin_enabled_.load_json(j["plugins"]);`. `LayoutCache::kVersion` is untouched (view_prefs.json is schemaless/best-effort).

### C.2 Gate lives in the loader (table membership) [critique-fix]

The loader takes a `PluginGate = std::function<bool(std::string_view id, PluginKind)>` consulted **once per discovery** (passed by `const&`, no per-node `std::function` copy). A disabled plugin is parsed/validated (its ops populate the panel diag) but **nothing reaches the Registry** — for declarative via the existing `register_into` arg (`Manifest.cpp:315`); for WASM by simply not calling `register_op_handler`/`register_parser`/`register_pass`. `discover_and_load_plugins` gains gate + `root` (testable) overloads; `LoadedManifest` gains `kind`, `id`, `enabled`, `registered`.

### C.3 Trust model + reload

- WASM discovered → appears in panel; registered only if explicit `true` **and** `WasmEngine::instance().enabled()`. Enabling requires a one-time confirm dialog; once persisted, no re-confirm.
- `Registry` gains **one additive** primitive `void reset_to_builtins()` (`{ lock; table_ = make_default_table(); }`, `Registry.cpp:262/298` reuse). `App::reload_plugins()`: `reset_to_builtins()` → `discover_and_load_plugins(gate)` → force cost recompute.
- **In-flight worker race** [critique-fix]: `reload_plugins()` does not just reset; it **bumps the session/cost generation** so any in-flight `CostModel`/`ShapeInferenceExt` job observing the transient builtin-only or partially-registered table is **superseded** (its result never published), not merely recomputed. Snapshot `shared_ptr` keeps each table alive (no dangle); generation-supersede prevents an inconsistent published cost.
- `App::init` replaces the bare `discover_and_load_plugins()` (`App.cpp:208`) with `reload_plugins()`, after `load_prefs()` (`App.cpp:203`) has populated `plugin_enabled_`.

### C.4 PluginsPanel — deferred toggle (no UAF) [critique-fix]

`draw_plugins_panel` binds `const auto& manifests = loaded_manifests()` and iterates holding `const LoadedManifest& lm`. Calling `reload_plugins()` **inline** would do `manifest_cache() = out` (`Manifest.cpp:348`), freeing the vector under the live `lm` reference (UAF / OOB on shorter reload). **Fix:** never mutate `loaded_manifests()` while iterating. Record a deferred action (file-static `pending_toggle_id`/`pending_toggle_val`, mirroring the existing `pending_wasm_id` + modal pattern) and apply `plugin_enabled().set(...); save_prefs(); reload_plugins();` **exactly once AFTER** the per-row loop and the modal block, when no reference into the cache is live. Per row: enable checkbox (greyed for Rejected), amber `[wasm • sandboxed]` vs green `[declarative • safe]` badge, status text. WASM off→on opens the confirm modal ("runs sandboxed, memory/fuel-capped, no filesystem/network, cannot read tensor bytes, but still arbitrary code — Enable?").

### C.5 File partition

**New:** `src/engine/plugin/PluginPrefs.{h,cpp}`; `tests/test_plugins_enable.cpp`.
**Edited:** `src/engine/plugin/declarative/Manifest.{h,cpp}` (gate plumbing + WASM discovery); `src/engine/plugin/Registry.{h,cpp}` (`reset_to_builtins()`); `src/engine/plugin/wasm/WasmHost.cpp` (`netvis_abi_version`/`netvis_op_abi_version` refusal in the run/load paths); `src/view/App.{h,cpp}` (`plugin_enabled_`, `reload_plugins()`, prefs); `src/view/PluginsPanel.cpp` (deferred toggle + modal + badges).

## Increment D — SDK header + `plugins/examples/` + ABI docs (closes #10 / #12)

### Scope

Ship the frozen authoritative wire ABI as a single freestanding C header, the authoring examples, the ABI doc, and the `static_assert`/layout bridge that keeps the SDK in sync with the C++ headers at host build time. The header was written at FREEZE (Part 1) and is shipped verbatim here.

### D.1 `plugins/sdk/netvis_plugin.h`

Single freestanding C11 header, `<stdint.h>` only. Two compile modes: **plugin build** (`__wasm__`) emits `extern` import decls + `NV_EXPORT` macros + a bump allocator; **host-bridge build** (plain C++ from `tests/test_sdk_abi.cpp`) emits only enums / wire structs / constants (imports `#if`'d out).

- **Version constants** mirroring the three C++ constants: `NETVIS_OP_ABI_VERSION`, `NETVIS_PARSER_ABI_VERSION`, `NETVIS_PASS_ABI_VERSION` (all `1u`).
- **Wire enums** exactly matching `ir::DType` (16 enumerators, `Unknown=15`) and `OpCategory` (`Other=14`); `nv_strid_t = uint32_t`.
- **Sandbox caps** mirroring `SandboxLimits`: `NV_MAX_MEMORY_PAGES`, `NV_MAX_STEPS`, `NV_NULL_GUARD_OFF` (8), plus the new caps `NV_MAX_RANK`, `NV_MAX_NODE_IO`, `NV_MAX_ATTR_LEN`, `NV_MAX_INTERN_LEN`, `NV_ERR_MSG_MAX`, `NV_SNIFF_HEAD`, `NV_SNIFF_TAIL`, `NV_READ_CHUNK_CAP`.
- **All 64-bit offsets/lengths/returns use `int64_t`/`uint64_t`, never `long`** (§0.5). **[critique-fix]**
- **Import decls** for the op facet (`"netvis_op"`, Increment A §A.2) and parser facet (`"netvis"`, Increment B §B.2) with exact `import_module`/`import_name` attributes; each carries its wasm3 sig string in a comment audited against the host `m3_LinkRawFunctionEx` literal.
- **`_Static_assert` on `sizeof`/`offsetof`** of every packed wire struct **inside the header itself** [critique-fix] so a packing divergence (i64/f64 alignment, tail padding) fails at compile on **both** the wasm32 plugin toolchain and the host toolchain — Part D's host-side bridge alone cannot catch guest-side packing.
