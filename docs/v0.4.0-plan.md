# NetVis v0.4.0 — Analyzer op-coverage + efficiency milestone (frozen plan)

> Status: **planned / contracts freezing**. This document is the authoritative
> spec the implementation phase builds against. It merges the 6-cluster design
> workflow output with the adjudicated fixes from the adversarial critique pass
> (every cluster came back `needs-revision`; the real defects the critics found
> are folded in below and marked **[critique-fix]**).

## Goal

"Support as many operations as possible to make graph heatmaps better and
efficiency calculation better." Concretely:

1. **Wider FLOP coverage** — elementwise/activation/reduce gap-fill, Attention,
   Recurrent (LSTM/GRU/RNN), Quantized (QLinear*/Integer/QDQ).
2. **New efficiency metrics** — per-node arithmetic intensity (FLOP/byte) and a
   roofline (memory-bound vs compute-bound) classification, model-level too.
3. **Selectable heatmap metric modes** — tint by FLOPs / Params / Activation
   bytes / Arithmetic intensity, not just log(FLOPs).

Everything upholds the existing contracts: **zero tensor-payload reads**,
**saturating uint64 arithmetic** (never overflow → clamp `UINT64_MAX`), **bounds-
checked** indices, **never throw**, and **honest-unknown** (unsupported op or
unresolved shape ⇒ `flops_known=false`, excluded from totals, never fabricated).

## The one decision the whole milestone hinges on — FLOP routing ownership

`OpCategory` today drives BOTH node color AND FLOP routing. That coupling is the
integration risk. **Frozen rule:**

- **Category owns COLOR only.**
- **FLOP correctness is owned by an explicit op-name dispatch that runs FIRST**
  in `compute_flops`, and each handler sets `flops`/`flops_known` and RETURNS
  before the category fallback.
- **Category-driven generic formulas run SECOND**, only for ops whose formula
  matches the generic assumption: `Activation|Norm|Elementwise ⇒ |O|`,
  `Reduce ⇒ |input0|`, `Pool ⇒ |O|·∏kernel_shape`, `MatMul(non-Gemm/Einsum) ⇒
  |O|·K`.
- **The `Quantize` category has NO generic fallback.** Every quantized op gets
  FLOPs only from an explicit handler, so e.g. `QLinearConv` (weight at input
  slot 3, not 1) can never accidentally be given an elementwise `|O|` number.
- **Adding an op to a color category never implies its FLOP formula.**
- Gap-fill ops (Sin/Cos/Erf/Softplus/…) get `|O|` for free purely by their
  `OpCategory` table entry → generic branch; **no `compute_flops` edit** for them.

`compute_flops` dispatch skeleton (after the existing `all_inputs_resolve` guard):

```
1. explicit op-name handlers (RETURN on match):
   Gemm; Conv/ConvTranspose (cat==Conv); MatMul (cat==MatMul && !Gemm && !Einsum);
   Einsum (constrained resolver, else unknown);
   Attention; MultiHeadAttention;
   LSTM; GRU; RNN;
   QLinearConv; ConvInteger; QLinearMatMul; MatMulInteger; QGemm;
   QLinearAdd; QLinearMul; QLinearGlobalAveragePool; QLinearAveragePool;
   QuantizeLinear; DequantizeLinear; DynamicQuantizeLinear
2. category fallback: Activation|Norm|Elementwise => |O|;  Reduce => |input0|;
   Pool => |O|*prod(kernel_shape);  MatMul(non-Gemm/Einsum) => |O|*K
3. else flops_known = false
```

> **[critique-fix]** design_item skeletons had dropped `Pool` from step 2 and
> forgotten `QGemm` from step 1 — both restored above.

Match op strings **normalized** inside `compute_flops` (lowercase, last dot-
segment) so `com.microsoft.QLinearConv` routes correctly — the existing branches
compare raw strings; new handlers normalize. (Reuse a small local `norm()` or
compare against `categorize_op` + a suffix check.)

## Per-op FLOP formulas (frozen, MAC = 2 FLOPs)

### Gap-fill (no `compute_flops` edit — OpCategory table only)
- **Elementwise `⇒ |O|`**: sin cos tan asin acos atan sinh cosh asinh acosh atanh
  erf reciprocal floor ceil round sign mod not xor bitwiseand/or/xor/not bitshift
  isnan isinf sum mean greaterorequal lessorequal.
- **Activation `⇒ |O|`**: softplus softsign hardswish mish logsoftmax celu
  thresholdedrelu shrink quickgelu.
- **Reduce `⇒ |input0|`**: reducel1 reducel2 reducelogsum reducelogsumexp
  reducesumsquare argmax argmin cumsum.
> **[critique-fix]** Coverage of gap-fill FLOPs is **contingent on shape
> inference** resolving the op's output/input shape. Many of these ops have NO
> shape handler today, so Agent C (shapes) MUST add the missing unary/variadic
> handlers (see below) or the `|O|` is silently unresolved. Sum/Mean are variadic
> — `|O|` is a **documented approximation** (true cost ≈ `(N-1)·|O|`); do not
> "fix" it inconsistently.

### Attention — fused `com.microsoft.Attention` (explicit handler)
input[0]=`[B,S,input_hidden]`, weights=input[1]=`[input_hidden, proj_out]`.
- `proj_MACs = B·S·input_hidden·proj_out`, `proj_out = weights.shape.back()`.
- **[critique-fix]** core hidden is the **attention** hidden, not input_hidden:
  `v_hidden = proj_out / 3` (QKV packed). Guard `proj_out % 3 == 0`; if not, the
  core term is honest-unknown (still count proj term, but mark whole node unknown
  if you can't get a coherent core — simplest: require divisibility, else
  `flops_known=false`).
- `core_MACs = 2·B·S·S·v_hidden` (QK^T + PV).
- `macs = proj_MACs + core_MACs` (safe_add); `flops = 2·macs`.
- Honest-unknown: input[0] not rank-3, weights missing/rank≠2, any dim<1,
  `proj_out%3≠0`.

### MultiHeadAttention — `com.microsoft.MultiHeadAttention` (explicit handler)
Pre-projected. query=input[0], key=input[1], value=input[2].
- `hidden_q` from query: rank3 → `shape[2]`; rank4 `[B,S,H,d]` → `shape[2]*shape[3]`.
- `S_kv` from key (rank3 `shape[1]`); fallback `S_kv=S_q` (self-attn).
- `macs = 2·B·S_q·S_kv·hidden_q`; `flops = 2·macs` (= `4·B·S_q·S_kv·hidden_q`).
- Head dim genuinely cancels here (pre-projected) → num_heads not needed.
- Honest-unknown: query rank ∉ {3,4}, any of B/S_q/hidden_q<1.

### Recurrent LSTM/GRU/RNN (explicit handler)
X=input[0]=`[seq,batch,input_size]`; W=input[1]; R=input[2]. `gates = 4|3|1`.
- `num_directions` = 2 if `direction=="bidirectional"` else (W rank3 → `W.shape[0]`)
  else 1.
- `hidden` = `hidden_size` attr if >0, else `R.shape[2]` (rank3), else `W.shape[1]/gates`.
- `per_step_per_dir = batch·gates·hidden·(input_size+hidden)`.
- **[critique-fix]** `macs = seq · num_directions · per_step_per_dir` (num_directions
  was missing → bidi half-counted). `flops = 2·macs`. All chained via `safe_mul`.
- Honest-unknown: X not rank-3, any of seq/batch/input_size<1 (dynamic seq is
  common → honestly unknown), hidden unresolvable.

### Quantized (explicit handlers, no generic fallback)
- **QLinearConv**: weight = input[**3**] `[Cout,Cin/g,k...]`, base = output[0].
  `macs = |O|·weight.shape[1]·∏(weight.shape[2..])`; `flops=2·macs`.
- **ConvInteger**: weight = input[**1**] (float-analogue slots). Same Conv math.
- **QLinearMatMul**: A=input[0], B=input[3]. `K = A.shape.back()`, `macs=|O|·K`.
- **MatMulInteger**: A=input[0], B=input[1]. `macs=|O|·K`.
- **QGemm** (`com.microsoft`): A=input[0], B=input[3]. `K = transA? A.shape[0] :
  A.shape.back()` (mirror the Gemm transA fix). `macs=|O|·K`, `flops=2·macs`.
- **QLinearAdd / QLinearMul** → `|O|` (elementwise).
- **QLinearGlobalAveragePool** → `|input0|` (global pool = one pass over input);
  **QLinearAveragePool** → `|O|·∏kernel_shape`.
- **QuantizeLinear / DequantizeLinear / DynamicQuantizeLinear** → `|O|` (pointwise
  scale/round; DynamicQuantize's min/max pass is captured since `|O|==|input|`).
> **[critique-fix]** QGemm, QLinearAdd/Mul, QLinearGlobalAveragePool were
> categorized Quantize with no formula in the original design → would have
> reported unknown despite being real compute. Explicit handlers added above.

### Einsum — constrained resolver (explicit handler)
`eq = get_string_attr("equation")`. Support ONLY: exactly 2 operands, no `...`,
single-char labels, no intra-operand repeat. Map each operand label → its input
dim; require label-count == rank; shared-label dim conflict ⇒ unknown.
`macs = ∏(dim over the union of all LHS labels)`; `flops = 2·macs`. Everything
else (incl. empty equation) ⇒ `flops_known=false` (preserves the existing "Einsum
not fabricated" regression test, which has no equation attr).

## Shape inference handlers (Agent C — `ShapeInferenceExt.cpp`)

FLOPs need resolved shapes. Best-effort, mutate-in-place, never crash, honest-
unknown. Dispatch on bare op string (parser drops domain). New internal helpers:
`attr_string` (for `direction`/`layout`), and factor `conv_output_shape(...)` /
`matmul_output_shape(a,b)` out of the existing Conv/MatMul branches (behavior-
identical — regression risk against v0.2.1 frozen code, keep byte-identical).

- **Attention**: output0 = copy input[0] shape+dtype (best-effort, **not** claimed
  exact); present_key/value dtype-only.
- **MultiHeadAttention**: output0 = `[B, S_q, v_hidden]`; `v_hidden` = value rank3
  `.back()`, **[critique-fix]** value rank4 `[B,N,S,H]` → `value[1]*value[3]`,
  else query `.back()`.
- **LSTM/GRU/RNN** (one parameterized branch, gate G=4 emits Y_c): Y=`[seq,dirs,
  batch,hidden]`, Y_h/Y_c=`[dirs,batch,hidden]`; **[critique-fix]** read
  `layout` attr — if `layout==1`, X=`[batch,seq,input]`, Y=`[batch,seq,dirs,
  hidden]`, Y_h/Y_c=`[batch,dirs,hidden]`. hidden from attr or R/W; dirs from
  `direction`/W. Partial `-1` dims allowed; hidden-unresolvable ⇒ dtype-only.
- **Quantized**: QLinearConv (`conv_output_shape` on slots 0/3, out dtype = y_zp
  slot7 or x), ConvInteger (slots 0/1, out dtype `I32`), QLinearMatMul
  (`matmul_output_shape` slots 0/3), MatMulInteger (slots 0/1, out `I32`), QGemm
  (Gemm shape slots 0/3), QLinearAdd/Mul (broadcast slots 0/3), QLinearGlobal/
  AveragePool (pool math, quant out dtype).
- **QDQ**: DequantizeLinear (shape-preserving, dtype → scale-slot1 dtype or F32),
  QuantizeLinear (shape-preserving, dtype → y_zp slot2 or U8),
  DynamicQuantizeLinear (out0 shape=input dtype U8; scale F32 scalar; zp U8 scalar
  — scalars dtype-only).
- **Gap-fill shapes**: add to the unary shape-preserving list: CumSum, Sinh, Cosh,
  Tan, Asin, Acos, Atan, Asinh, Acosh, Atanh, Mish, ThresholdedRelu, Celu, Shrink,
  QuickGelu, Reciprocal-family already present; **Not** → shape-preserving dtype
  `Bool`. **[critique-fix]** add **Xor** to the binary-broadcast list (dtype
  `Bool`, alongside And/Or/Equal) — it was omitted.
- **[critique-fix]** variadic **Sum/Mean**: emit a shape ONLY when ALL inputs
  resolve (fold `broadcast` across all); any unresolved input ⇒ dtype-only. (An
  N-ary fold over only the resolved subset can emit a too-small fabricated shape.)

## Heatmap metric modes (Agent D — view)

Single extractor `metric_value` feeds all three consumers (range/tint/legend) so
they cannot diverge (the v0.3.1 group-scale bug class).

- `recompute_heatmap_range`: per display node, `sum_node_costs` → `metric_value`
  → skip `!known || value==0`, track min/max into the (now `double`) range cache.
- `cost_tint_for_display`: gray gate keys on `metric_value(...).known` for the
  **selected** metric (not always `flops_known`); else normalize + ramp.
- `draw_heatmap_legend`: title = `heatmap_metric_name(m)`; **[critique-fix]** min/
  max rendered by a metric-aware `format_metric_value` (FLOPs→`human_flops`,
  Params→`grouped_count+" params"`, ActBytes→`human_bytes`, AI→`"%.2f F/B"` after
  `/kArithIntensityScale`) — `human_flops` alone would mislabel bytes as "MF".
- `normalize_flops` → renamed `normalize_metric_value` (body unchanged, already
  metric-agnostic integer log/linear). **[critique-fix]** also update the
  `heatmap_range()` body (CostPanel.cpp:~207) to the renamed `min_value`/`max_value`
  fields — the rename touches three sites, not two.
- **[critique-fix]** `sum_node_costs` MUST additionally sum `input_act_bytes`
  (and any other new NodeCost field) or collapsed-group AI is silently wrong.
- Metric combo in `draw_cost_section` (inside `if (vs.cost_heatmap)`); sets
  `dirty` → existing `save_prefs`. One-frame stale range on metric switch is
  acceptable (self-heals next frame).
- Prefs: new string key `heatmap_metric` (save via `heatmap_metric_name`, load via
  `is_string` guard + `heatmap_metric_from_name` → default Flops). Backward-
  compatible; missing key → Flops. **No LayoutCache kVersion bump** (tint is not

- Roofline panel: `draw_cost_section` shows model `overall_arithmetic_intensity`,

## Known coverage gaps / non-goals (documented honestly)

- **Decomposed attention** (bare MatMul+Softmax+MatMul) is NOT pattern-detected —
  its constituent nodes already get FLOPs from the MatMul/Activation handlers;
  detecting it would double-count.
- **`MatMulNBits`, `QAttention`, `GroupQueryAttention`, `SkipLayerNormalization`,
  `EmbedLayerNormalization`, `DynamicQuantizeMatMul`, `MatMulIntegerToFloat`** —
  out of scope for v0.4.0 (fall to `Other`/unknown, honestly). Candidates for a
  follow-up.
- **Dynamic seq/batch recurrent + dynamic-shape graphs** → honest `flops_known=
  false`; surfaced in the existing "FLOPs unknown k/N nodes" line.
- **Roofline ridge presets** are datasheet estimates, not measurements — the panel
  must label roofline output "approximate (machine-balance relative)".
- **View cache-keying path stays GUI-only** (no core-only test harness) — verified
  via the xvfb smoke run, as in prior milestones.
